# apps

End-user applications and platforms — a pan-India services platform, a banking system, a chat app, an Excel tool, and an AI health coach.

Each linked doc below was written by **reading the actual source**, not the README — with computed line counts and an explicit "README vs. code" section. Several projects here have a large gap between the README's pitch and what the committed code actually does.

| Project | What it actually is | Stack / size | Notable code-level finding |
|---------|---------------------|--------------|----------------------------|
| [SARVA](SARVA.md) | A **pan-India life-utility platform** ("sab kuch, sabke liye") — currently in foundation phase | Go + Flutter · ~2,883 Go + ~2,798 Dart LOC | One real Go service (`core-api`: phone-OTP auth + `/v1/me` CRUD) + a Flutter auth shell. The auth is genuinely strong — **RS256 JWT with refresh rotation, argon2id OTP, and real Postgres Row-Level Security** on all 5 tables. But the **four pillars (Samay/Parivar/Adhikar/Awaaz) and the geo/realtime/ai microservices exist only in `docs/`**; MQTT, pgvector, OSM, mTLS and any LLM are absent. `REDIS_URL` is required config but no Redis client is ever opened. README is candid about being foundation-phase. |
| [BankSystem](BankSystem.md) | A **console banking system** in C with a live stock-market simulation | C · 8,466 LOC (excl. dead archive) | "~15,000 lines" is overstated by ~50%, and it's a **console** app (Win32 console API + ANSI), not a GUI. One worker thread (a 20s stock ticker) races shared state with no locking; "multi-session" is file spin-lock inter-process coordination. Absolute-model price sim, no order book; plaintext 4-digit PIN + no-op "2FA"; flat-file persistence. The `.vcxproj` references nonexistent files, so it **won't build unedited**. |
| [JavaChat](JavaChat.md) | A **WhatsApp-style chat clone** with a custom TCP protocol | Java · Swing · MySQL · 4,192 LOC | Line-oriented wire protocol `COMMAND::p1::p2|||` (README wrongly says `:::`); genuine thread-per-client server. But it **won't compile as committed** — `WhatsAppServer`/`ClientHandler`/`MessageRouter` import non-existent `whatsapp.common.*` packages. One shared JDBC connection across all threads (README's "connection pooling" is false); passwords stored in plaintext despite a `password_hash` column. |
| [ExcelLinker](ExcelLinker.md) | A **Tkinter GUI** that joins Excel/CSV sheets via pandas `merge`/`join` | Python · pandas + openpyxl · 5,874 LOC | The shipped "current" v6.0 (1,753 lines) has its **core join engine stubbed out** — `start_build`/`rebuild_dataset` are "next phase" placeholders, `grep pd.merge` returns nothing, so a user can't actually produce a merged file. The real, working join logic survives only in the three `archive/` versions (v1–v3). |
| [Personal-Health-Assistant](Personal-Health-Assistant.md) | A **full-stack AI nutrition & fitness coach** (not a symptom checker or diagnosis model) | Flask + React/Vite · 2,674 Py + 1,147 JS LOC | The only AI is the **Cerebras Cloud SDK calling Llama-4-Scout** (`llama-4-scout-17b-16e-instruct`); no trained model or dataset. SQLite + 9 SQLAlchemy models; JWT + bcrypt auth. README advertises badges and streaks, but **achievements are never awarded and streaks are always 0** (the relevant functions are dead code); the Workout model is vestigial. |
